PulseCheck
==============

**AI-Powered Infrastructure Observability for Azure**

PulseCheck is a lightweight, production-ready monitoring API built with **NestJS**. It doesn't just ping your services; it uses **OpenAI** to perform deep-packet analysis of HTTP headers and latency, providing actionable insights into infrastructure bottlenecks before they become outages.

 Why This Project?
--------------------

Most uptime monitors tell you *if* a site is down. PulseCheck tells you *why* it's slow. By deploying this to **Azure App Service**, I've engineered a solution that demonstrates:

-   **PaaS Orchestration:** Managed deployment on Azure App Service (Linux).

-   **AI-Driven SRE:** Utilizing LLMs for automated log and header analysis.

-   **Enterprise CI/CD:** Zero-downtime deployments using GitHub Actions and Azure Deployment Slots.

* * * * *

Tech Stack
--------------

-   **Backend:** NestJS (Node.js)

-   **AI Engine:** OpenAI GPT-4o-mini

-   **Cloud:** Microsoft Azure (App Service, Key Vault)

-   **CI/CD:** GitHub Actions

* * * * *

 Architecture
----------------

1.  **Ingress:** A RESTful endpoint receives a target URL.

2.  **Telemetry:** The NestJS service performs a high-precision `GET` request, capturing TTFB (Time to First Byte), SSL handshakes, and full header sets.

3.  **Intelligence:** If anomalies are detected (latency > 500ms or 4xx/5xx errors), the data is piped to OpenAI.

4.  **Insight:** The API returns a JSON report including a "Root Cause Hypothesis" generated by the AI.

Infrastructure & Deployment
------------------------------

This project is optimized for the **Azure Ecosystem**:

Environment Variables
---------------------

PulseCheck requires the following secrets, managed via **Azure App Service Configuration**:

Code snippet

```
PORT=8080
OPENAI_API_KEY=your_key_here
NODE_ENV=production

```

CI/CD Pipeline
--------------

Every push to `main` triggers a GitHub Action that:

1.  Lints and builds the NestJS application.

2.  Deploys to an **Azure Deployment Slot** (`staging`).

3.  Performs an automated health check.

4.  **Swaps** the slot to `production` for zero-downtime updates.

Getting Started
------------------

Prerequisites
-------------

-   Node.js v20+

-   An active Azure Subscription (Free/Student credits work perfectly)

Local Development
-----------------

1.  Clone the repo: `git clone https://github.com/Nabil201-ctrl/PulseCheck.git`

2.  Install deps: `npm install`

3.  Start the dev server: `npm run start:dev`

API Usage
---------

Bash

```
curl -X POST https://pulsecheck-app.azurewebsites.net/check\
-H "Content-Type: application/json" \
-d '{"url": "https://example.com"}'
```
